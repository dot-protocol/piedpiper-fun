dotdrop

Drop a file. Share a hashtag. No accounts. No servers holding your data.

what it is

dotdrop is a peer-to-peer encrypted file transfer tool built on the Pipernet protocol. A file leaves your device already encrypted — the server never sees the plaintext. The hashtag is the key.

It is part of the axxis.world identity layer. Every transfer is signed with an Ed25519 keypair derived from your device. No account creation. No login. Your identity is your keypair.

how to use it

1
Open the app. Go to dotdrop.axxis.world on the sending device.
2
Drop a file and type a #hashtag. The hashtag is the shared secret. It never leaves the browser unencrypted.
3
Share the hashtag out-of-band. Text it, say it aloud, write it on paper. The channel is piedpiper.fun/dotdrop/#your-tag.
4
Receiver opens the same URL. WebRTC connection is negotiated via Nostr relays. File transfers directly between devices.

technical design

Signaling uses Nostr relays — no custom signaling server. Each DropPeer generates an ephemeral Nostr keypair per session. Offers, answers, and ICE candidates are addressed by Nostr pubkey and relayed through public infrastructure.

Files are encrypted with AES-256-GCM before leaving the device. The key is derived from the hashtag using HKDF-SHA256. The Nostr relay sees only opaque ciphertext addressed to a pseudonymous public key.

transportWebRTC DataChannel (direct P2P)
signalingNostr relays (ephemeral keypairs)
encryptionAES-256-GCM, key from HKDF-SHA256(hashtag)
integritySHA-256 of plaintext, verified on receive
identityEd25519 (signing), X25519 (DM encryption)
storageIndexedDB, local only
servernone — no file data touches a server
accountsnone — keypair is your identity

canonical dot format

Every transfer is recorded as a canonical DOT — a fixed 153-byte signed observation. Each DOT chains to the previous via SHA-256, forming a tamper-evident local history. The format is identical to the one used across all axxis.world identity operations.

// 153 bytes, fixed-width — enables O(n) chain traversal [0..31] pubkey 32B Ed25519 public key [32..95] sig 64B Ed25519 signature over signed bytes (89B) [96..127] chain 32B SHA-256(prevDotBytes), zeros for genesis [128..135] ts 8B Unix ms, big-endian int64 [136] type 1B 0x00=public 0x01=circle 0x02=private 0x03=ephemeral [137..152] payload 16B zero-padded context (sha256(content)[:16] for drops)

identity layer (axxis.world)

dotdrop inherits the three-keypair identity model from axxis.world. Keys are generated once per device and stored in IndexedDB — never exported, never transmitted.

keypair 1
Ed25519
DOT signing. Canonical identity. Used for all observations and transfer records.
keypair 2
secp256k1
Nostr protocol. Used for signaling relay messages. Ephemeral per DropPeer session.
keypair 3
X25519
ECDH shared secret for direct message encryption. AES-256-GCM derived from Diffie-Hellman exchange.
